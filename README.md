# 系统性能日志解析与报告生成工具

## 项目简介

本工具用于解析系统性能日志，提取内存、CPU、交换空间以及百度导航进程的性能数据，并自动生成Excel数据文件和交互式HTML报告。

## 功能特性

1. **日志解析**：自动解析系统性能日志文件，提取关键性能指标
2. **数据存储**：将解析后的数据保存为Excel文件，包含多个工作表
3. **自动报告生成**：解析完成后自动生成交互式HTML报告
4. **图表可视化**：使用折线图展示性能趋势，支持数据系列的显示/隐藏切换
5. **颜色区分**：为不同数据系列使用明显不同的颜色，提高可读性

## 目录结构

```
系统性能数据解析/
├── 日志/                  # 存放待解析的系统性能日志文件
│   └── sys_perf_*.log    # 系统性能日志文件
├── output/               # 输出目录
│   ├── *.xlsx            # 生成的Excel数据文件
│   └── *.html            # 生成的HTML报告文件
├── parse_perf_log.py     # 日志解析主脚本
├── generate_interactive_report.py  # 交互式报告生成脚本
├── generate_html_report.py         # 静态报告生成脚本
└── check_excel_time.py   # 时间格式检查脚本
```

本工具需要以下Python依赖包：

- pandas: 数据处理


## 使用方法

### 1. 准备日志文件

将需要解析的系统性能日志文件放入`日志/`目录中，日志文件格式应为系统性能监控工具生成的标准格式。

### 2. 运行日志解析脚本

```bash
python3 parse_perf_log.py
```

该脚本会自动：
- 读取`日志/`目录下所有`.log`文件
- 解析每个日志文件，提取性能数据
- 将数据保存到`output/`目录下对应的Excel文件中
- 为每个解析的日志文件自动生成交互式HTML报告

### 3. 查看报告

解析完成后，在`output/`目录下可以找到生成的Excel数据文件和HTML报告文件。使用浏览器打开HTML文件即可查看交互式性能报告。

## 数据指标说明

### 内存指标

- **已用内存(M)**：系统当前已使用的内存量（MB）
- **可用内存(M)**：系统当前可用的内存量（MB）
- **缓冲区(M)**：用于文件系统缓冲区的内存量（MB）
- **百度导航内存(M)**：百度导航进程使用的内存量（MB）

### CPU指标

- **CPU总使用率(%)**：CPU总体使用率
- **用户态CPU(%)**：用户态进程使用的CPU百分比
- **Nice CPU(%)**：低优先级用户态进程使用的CPU百分比
- **系统态CPU(%)**：内核态进程使用的CPU百分比
- **空闲CPU(%)**：空闲CPU百分比
- **IO等待(%)**：等待IO操作的CPU百分比
- **硬中断(%)**：处理硬中断的CPU百分比
- **软中断(%)**：处理软中断的CPU百分比
- **Host CPU(%)**：在虚拟化环境中，宿主机的CPU使用率
- **百度导航CPU(%)**：百度导航进程使用的CPU百分比

### 交换空间指标

- **已用交换空间(M)**：已使用的交换空间量（MB）
- **可用交换空间(M)**：可用的交换空间量（MB）
- **缓存(M)**：交换空间缓存量（MB）

## 常见问题

### 1. 为什么百度导航的性能数据为0？

可能原因：
- 日志文件中没有包含百度导航进程的相关数据
- 百度导航进程的名称与预期不符（预期名称：com.baidu.naviauto或com.baidu.navia）

### 2. 图表颜色太相似怎么办？

系统已经为不同数据系列设置了明显不同的颜色：
- 已用内存：蓝色
- 可用内存：红色
- 缓冲区：绿色
- 百度导航内存：紫色
- 用户态CPU：蓝色
- 百度导航CPU：红色

### 3. 报告中时间刻度不正确？

系统使用日志文件中记录的时间戳，时间精度为秒级。如果时间显示不正确，可能是日志文件中的时间格式问题。
